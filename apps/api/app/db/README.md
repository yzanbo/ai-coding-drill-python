# db/

## db/ とは何か

DB への接続を作り、リクエストごとに「**セッション**」を貸し出すフォルダ。

「接続」と「セッション」は別物：

- **AsyncEngine（接続プール）**：アプリ全体で 1 つだけ作る、DB との物理的な配線。コネクションを再利用する箱
- **AsyncSession（1 リクエスト 1 つの会話チャネル）**：エンジンから 1 本借りる。1 リクエスト = 1 セッション = 1 トランザクション境界
- **Base**：SQLAlchemy モデル共通の親クラス（[models/](../models/) のクラスはこれを継承）

実際の SQL は [services/](../services/) 側で書く。ここは「アプリと Postgres の**通信路（出入り口）**」を用意する場所。SQL の送信（クエリ）と結果の受信（rows）の両方向がこの窓口を通る。
セッションを router に届ける時は [deps/](../deps/) の `get_async_session` 経由で取り出す
（router が直接ここを import するのは NG）。

## 中身

- `session.py`：DB との接続を作り、リクエストごとに 1 つセッションを貸し出す関数
- `base.py`：SQLAlchemy のモデル共通の親クラス（[models/](../models/) のクラスはこれを継承）
