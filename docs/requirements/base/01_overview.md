# 01. プロジェクト概要

## プロジェクト名（仮）
AI Coding Drill（仮称）

## 一言説明
LLMが自動生成したプログラミング問題を、サンドボックス環境で検証・採点する学習サイト。初期は TypeScript / Node.js から開始し、将来的に複数言語・フレームワークへ拡張する。

## 背景・動機
- 既存の学習サイト（Paiza、AtCoder等）は問題数が固定で、学習者の弱点に合わせた出題が難しい
- LLM単体で問題生成しても、動作しないコードやテストケースの誤りが混入する
- 「LLMの出力を信用せず、サンドボックスで動作保証する」仕組みを示すことで、実務的なAI活用設計力をアピールする

## ゴール
1. 学習者が自分の弱点に合わせた問題を無限に解ける
2. 生成された問題は必ずサンドボックスで動作検証済み
3. 技術選定・設計判断を README / アーキテクチャ図で語れる状態にする

## ターゲットユーザー
- プログラミング中級者（基本文法は分かるが、実戦経験が少ない層）
- 対応言語：初期は TypeScript（Node.js 環境、JavaScript も受け入れ可）
- 拡張ロードマップ：Python、Next.js（React コンポーネント採点）など、将来的に他言語・フレームワークを追加できる設計にする

## 問題カテゴリ方針
- アルゴリズム・データ構造
- 文字列処理・配列操作
- 非同期・エラーハンドリング（Promise / async）
- 型パズル（TypeScript ならではの差別化軸）
- その他、汎用的な基礎課題を幅広くカバー

## ポートフォリオとしての狙い
- **差別化軸**：問題生成 × サンドボックス検証 × 品質保証パイプライン
- **技術アピール範囲**：バックエンド設計、コンテナ・セキュリティ、LLMアプリ設計、観測性、ポリグロット構成（TS + Go、次バージョンで Python）
- **対象ポジション**：バックエンド / AIエンジニア / SRE寄り

## 言語・フレームワーク構成ロードマップ
- **MVP**：Next.js（フロント）+ **NestJS**（API・LLM）+ **Go**（採点ワーカー）
- **次バージョン**：Python（評価・分析パイプライン、RAG 等）を追加
- **将来**：採点対象言語の多言語化（Python、Next.js コンポーネント採点 等）

## スコープ外
- モバイルアプリ
- 有料課金機能
- 複数ユーザーのリアルタイム対戦
