# `01-overview.md` 用テンプレート

[01-overview.md](./01-overview.md) を 0 ベースから作成する時、または既存ファイルを大幅に書き換える時に使う雛形。

`01-overview.md` は **「プロジェクトの存在意義」** を扱う。具体的には：

- プロジェクト名・一言説明
- 背景・動機（なぜ作るのか）
- ゴール
- 問題カテゴリ方針 / 対象領域
- ポートフォリオとしての狙い（または対象市場）
- スコープ外

ペルソナ・ユーザーストーリーは別ファイル（[`_template-02-personas.md`](./_template-02-personas.md) / [`_template-03-user-stories.md`](./_template-03-user-stories.md)）の領分。

---

## 使い方

1. このファイルの「テンプレ本体」セクションをコピー
2. [01-overview.md](./01-overview.md) に貼り付け
3. `{...}` の placeholder を埋める
4. 不要なセクションは削除（必須セクションは ✅ マーク）

---

## 守備範囲の遵守チェック（書く前に）

- [ ] このファイルに書く内容は **「変わらない部分」** か（変わりやすいなら他バケットを検討）
- [ ] 機能個別の受け入れ条件・画面詳細を含めていないか → [4-features/](../4-features/) へ
- [ ] 技術選定の具体名・ライブラリ名を含めていないか → [2-foundation/05-runtime-stack.md](../2-foundation/05-runtime-stack.md) へ
- [ ] アーキテクチャの責務記述を含めていないか → [2-foundation/02-architecture.md](../2-foundation/02-architecture.md) へ
- [ ] ロードマップ・期間を含めていないか → [5-roadmap/01-roadmap.md](../5-roadmap/01-roadmap.md) へ

## 記入時の Tips

- **「一言説明」は 1 文で書ききる**：複数文になるなら背景・動機セクションへ
- **「ゴール」は 3 つ程度に絞る**：多すぎると優先度が曖昧になる。残りはスコープ外か将来計画へ
- **「スコープ外」を明示することは重要**：「やらないこと」が明確だと、議論時に「これはスコープ内か？」が判断しやすい
- **「ポートフォリオとしての狙い」は受託・社内ツールでは削除可**：プロジェクト性質に応じてセクション自体を残すかどうか判断
