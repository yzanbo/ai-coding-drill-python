# 1-vision/

**変更頻度：極小**（プロジェクト方針再定義時のみ更新）

---

## このディレクトリの役割

プロジェクトの**変わらない部分**を定義する。

- プロジェクトのビジョン・ゴール・差別化軸
- 想定利用者（ペルソナ）
- ペルソナ別ユーザーストーリー（ストーリーは機能追加時に追記される）

このディレクトリは **「なぜこのサービスを作っているか」** に答える。実装詳細・技術選定・ロードマップは扱わない（他バケットを参照）。
