# `02-personas.md` 用テンプレート

[02-personas.md](./02-personas.md) を 0 ベースから作成する時、または既存ファイルに**新しいペルソナを追加する時**に使う雛形。

`02-personas.md` は **「誰がこのサービスを使うのか」** を扱う。属性・利用文脈・関係する機能を定義する。

ユーザーストーリー（ペルソナが何をしたいか）は別ファイル（[`_template-03-user-stories.md`](./_template-03-user-stories.md)）の領分。

---

## 使い方

### ケース 1：0 ベースから 02-personas.md を作る時

1. 「ファイル全体テンプレ」セクションをコピーして [02-personas.md](./02-personas.md) に貼り付け
2. 「ペルソナ単位テンプレ」を必要な数（通常 2〜3 個）コピーして埋める
3. 末尾の「ペルソナの優先度表」を埋める

### ケース 2：既存 02-personas.md に新ペルソナを追加する時

1. 「ペルソナ単位テンプレ」セクションをコピー
2. [02-personas.md](./02-personas.md) の適切な位置に挿入し `{...}` placeholder を埋める
3. 末尾の「ペルソナの優先度表」に 1 行追加
4. [03-user-stories.md](./03-user-stories.md) にも対応するセクションを追加（[`_template-03-user-stories.md`](./_template-03-user-stories.md) 参照）

## 守備範囲の遵守チェック（書く前に）

- [ ] 既存ペルソナで吸収できないか確認（属性・利用文脈が本質的に違うか）
- [ ] そのペルソナ向けの機能・要件が **実際に発生するか**（思考実験だけで終わらないか）
- [ ] 利用文脈に **「いつ・どこで・どのデバイスで・何分くらい」** が書けるか
- [ ] 機能個別の受け入れ条件を書いていないか → [4-features/](../4-features/) へ


## ペルソナ単位テンプレ（追加時のメイン雛形）

```markdown
## ペルソナ {識別子 A/B/C/...}：{ペルソナ名}

**{主要 / 副次}ペルソナ**。{1 行で立場・役割を補足}。

### 属性

- {年齢層・職業・経験年数 など、設計判断に効く属性}
- {利用前提（GitHub アカウント所有 / 特定 OS / 特定環境）}
- {モチベーション・課題意識}

### 利用文脈

- 利用シーン：{いつ・どこで使うか}
- 利用デバイス：{PC / スマホ / 両方}
- セッションあたり：{1 回の利用で何をどれくらいするか}
- 期待する体験：{特に重視する UX 要素}

### 関係する機能

- [F-XX: 機能名](../4-features/F-XX-...md)
- F-YY: 機能名（バックログ、[5-roadmap](../5-roadmap/01-roadmap.md#f-yy-...))

### 関係する横断テーマ（任意）

<!-- 観測性・非機能・アーキテクチャ など、特定ペルソナと結びつく領域。なければ削除 -->

- {観測性 / 非機能 / アーキテクチャ など}
```

---

## 記入時の Tips

- **属性は「設計判断に効くもの」のみ**：「30 代男性」のような市場調査属性ではなく、「GitHub アカウント所有」「型診断のある言語経験者」のように **要件決定に直結する属性**を書く
- **利用文脈は「設計者が想像で書ける」レベルではダメ**：実際のユーザーインタビュー or 自分自身がそのペルソナと一致する場合のみ書ける。書けないなら「わからない部分」と明示
- **関係する機能は変動する**：機能追加時に追記。ペルソナ定義自体は変わらないが、結びつく機能は増減する
- **副次ペルソナは多くしない**：3 つを超えると焦点がぼやける。本当に独立したペルソナか、既存の派生ではないかを再考する
