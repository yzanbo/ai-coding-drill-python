## ペルソナ A：プログラミング学習者（中級）

**主要ペルソナ**。本サービスのコアユーザー。

### 属性

- 基本文法を理解しているが実戦経験が少ない層
- GitHub アカウントを持っている前提（OAuth でログイン）
- 自分の弱点を客観的に把握したいモチベーションがある
- 対応言語：MVP は TypeScript（Node.js 環境）、将来 Python 等へ拡張

### 利用文脈

- 学習時間：通勤・休日・スキマ時間
- 利用デバイス：PC（コードエディタを使うため）
- セッションあたり：1〜数問を解いて結果を確認
- 期待する体験：解答を即座に採点・フィードバックして欲しい、過去履歴で進捗が見たい

### 関係する機能

- [F-01: GitHub OAuth ログイン](../4-features/F-01-github-oauth-auth.md)
- [F-02: 問題生成リクエスト](../4-features/F-02-problem-generation.md)
- [F-03: 問題表示・解答入力](../4-features/F-03-problem-display-and-answer.md)
- [F-04: 自動採点](../4-features/F-04-auto-grading.md)

- F-07: LLM ヒント機能（バックログ）

## ペルソナ B：サービス運用者（自分自身）

**ポートフォリオ運用ペルソナ**。プロジェクトを公開・維持する立場。

### 属性

- 本プロジェクトの作者（個人開発）
- 採用活動の文脈でポートフォリオとして公開
- 24/7 オンコール体制ではないが、コスト超過や障害には早期対応したい

### 利用文脈

- 観測ダッシュボードで定期的にサービス状態を確認
- アラートが発火した時に Runbook で原因切り分け・対応
- LLM 出力の品質指標（Judge スコア・ユーザー行動シグナル）を継続観測

### 関係する機能・横断テーマ

- F-08: 管理ダッシュボード（バックログ、[5-roadmap/01-roadmap.md](../5-roadmap/01-roadmap.md#f-08-管理ダッシュボード)）
- 観測性：[2-foundation/04-observability.md](../2-foundation/04-observability.md)
- 非機能要件：[2-foundation/01-non-functional.md](../2-foundation/01-non-functional.md)
- LLM パイプライン：[2-foundation/03-llm-pipeline.md](../2-foundation/03-llm-pipeline.md)

## ペルソナ C：採用担当者・面接官（外部閲覧者）

**評価ペルソナ**。ポートフォリオを評価する立場。

### 属性

- ポートフォリオサイトから本サービスへのリンクを辿る
- 限られた時間で「設計力・運用力を持つエンジニアか」を判断したい
- 技術判断の根拠（なぜこの技術を選んだか）に関心がある

### 利用文脈

- 短時間で全体構造を把握したい：README → 設計判断（ADR） → 実装の流れ
- サービスを実際に触って動作確認
- ダッシュボードで観測性が機能していることを確認
- 設計に関する質問を面接で投げかける材料を集める

### 関係するドキュメント

- リポジトリ TOP の [README.md](../../../README.md)
- 要件定義書（本ディレクトリ群）
- ADR（[docs/adr/](../../adr/)）
- 公開デプロイされたサービス本体（R5 完了後）

| ペルソナ | 優先度 | 設計判断時の重み |

| A：学習者 | **最高** | UX・機能設計の主軸 |
