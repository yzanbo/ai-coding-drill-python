# F-03: 問題表示・解答入力

## ユーザーストーリー

As a **ゲスト / 認証ユーザー（プログラミング学習者）**
I want **問題一覧から興味のある問題を選び、コードエディタで解答を書いて実行できる**
So that **実際にコードを書いてプログラミング能力を試したいから**

## 受け入れ条件（Definition of Done）

- [ ] `/problems` で問題一覧（タイトル / カテゴリ / 難易度）が表示される
- [ ] カテゴリ・難易度でフィルタできる
- [ ] ゲストでも問題一覧・問題詳細を閲覧できる
- [ ] `/problems/:id` で問題詳細（問題文・入出力例）が表示される
- [ ] 問題詳細画面の **テストケースの一部はマスクされる**（全テストケースを見せると採点が無意味になるため）
- [ ] コードエディタで TypeScript コードを入力できる
- [ ] エディタは**インライン型診断・補完**が効く（→ [05-runtime-stack: フロントエンド](../2-foundation/05-runtime-stack.md#フロントエンド)）
- [ ] 「実行」ボタン押下で解答を送信できる（**解答送信は認証必須**）
- [ ] ゲストが「実行」を押すと `/login` にリダイレクトされる
- [ ] 解答送信後、採点結果は同画面で表示される（→ [F-04: 自動採点](./F-04-auto-grading.md)）

## 概要

問題の閲覧と解答入力に特化した画面群。閲覧はゲスト含む全員、解答送信は認証ユーザーのみ。コードエディタはサーバ採点前に**ブラウザ内で型診断・補完**を提供し、UX を向上させる。

## ビジネスルール

- **問題閲覧はゲストでも可**：ポートフォリオとして「触ってみる」体験を妨げないため
- **解答送信は認証必須**：履歴・統計の前提として認証が必要
- **テストケースの一部をマスク**：「入出力例」として一部を見せ、残りは隠す。完全公開すると LLM やコピペで簡単に通せてしまう
- **コードエディタはサーバ採点前にブラウザ内で型診断**：構文エラーレベルのコードがサーバに送られるのを減らし、サンドボックスコストを節約
- **エディタ内容はローカルストレージに自動保存**：誤遷移時の復元（任意機能、後述）

## スコープ外（このスプリントでは扱わない）

- 採点ロジック自体（[F-04: 自動採点](./F-04-auto-grading.md)）
- LLM ヒント機能（[F-07: LLM ヒント](../5-roadmap/01-roadmap.md#f-07-llm-ヒント機能)）
- 解答コードの SNS 共有
- お気に入り問題保存（必要性が出てから検討）
- 多言語対応（MVP は TypeScript のみ。言語アダプタ層で将来拡張）

## データモデル

詳細は [3-cross-cutting/01-data-model.md](../3-cross-cutting/01-data-model.md) を参照。

- `problems`：問題本体（`title`, `description`, `examples`, `test_cases`, `reference_solution`, `category`, `difficulty`）
- `submissions`：解答送信時に作成される（[F-04](./F-04-auto-grading.md) で詳細）

## 画面

### 問題一覧画面（対象：ゲスト / 認証ユーザー）

- **ルート**：`/problems`
- **概要**：問題のリスト + カテゴリ・難易度フィルタ
- **主要コンポーネント**：`<ProblemList />`、`<CategoryFilter />`、`<DifficultyFilter />`、`<PaginationControl />`
- **使用 API**：
  - `GET /problems?category=...&difficulty=...&page=...` — 一覧取得（フィルタ・ページネーション）
- **主要インタラクション**：
  - フィルタ変更でリスト再取得
  - 行クリックで問題詳細画面へ遷移

### 問題詳細・解答画面（対象：ゲスト / 認証ユーザー）

- **ルート**：`/problems/:id`
- **概要**：問題文・入出力例の表示 + コードエディタ + 実行ボタン
- **主要コンポーネント**：
  - `<ProblemDescription />`（問題文・入出力例）
  - `<CodeEditor />`（CodeMirror 6 ベース、ブラウザ内型診断）
  - `<RunButton />`（実行 = 解答送信）
  - `<GradingResult />`（[F-04](./F-04-auto-grading.md) で詳細）
- **使用 API**：
  - `GET /problems/:id` — 問題詳細取得（テストケース一部マスク）
  - `POST /submissions` — 解答送信（[F-04](./F-04-auto-grading.md) で詳細）
- **主要インタラクション**：
  - エディタ編集 → ローカル状態に保持、ローカルストレージに自動保存
  - 「実行」押下 → 認証チェック → 解答送信 → 採点ステータスポーリング

## ユーザーフロー

### 問題閲覧 → 解答送信フロー（対象：認証ユーザー）

1. ユーザーが `/problems` で一覧を見る
2. カテゴリ・難易度でフィルタ
3. 問題行をクリック → `/problems/:id` に遷移
4. 問題文・入出力例を読み、コードエディタで TypeScript コードを書く
5. ブラウザ内の型診断で構文エラー・型エラーを修正
6. 「実行」ボタン押下
7. `POST /submissions` で解答送信（→ [F-04](./F-04-auto-grading.md) のフロー）
8. 同画面でポーリングしながら採点結果を待つ
9. 結果表示（全通過 = 正解、一部失敗 = 失敗ケース表示）

### ゲストが解答送信を試みた場合

1. ゲストが `/problems/:id` で「実行」を押下
2. フロントが `GET /auth/me` で未認証を確認 → `/login?next=/problems/:id` にリダイレクト
3. ログイン後、元の問題画面に戻ってきて続きから解答送信可能

## API

| メソッド | パス | 用途 | 認証 |
|---|---|---|---|
| GET | `/problems` | 問題一覧（カテゴリ・難易度フィルタ可） | 任意 |
| GET | `/problems/:id` | 問題詳細（テストケースの一部はマスク） | 任意 |

機械可読の最新仕様は OpenAPI（`/api/docs/openapi.json`）が SSoT。`POST /submissions` は [F-04](./F-04-auto-grading.md) を参照。

## バリデーション

| フィールド | ルール | エラーメッセージ |
|---|---|---|
| `category` (Query) | 任意、許可値リスト内 | 不正なカテゴリです |
| `difficulty` (Query) | 任意、`easy` / `medium` / `hard` | 不正な難易度です |
| `page` (Query) | 任意、整数、1 以上 | ページ番号が不正です |
| `id` (Path) | UUID | 不正な問題 ID です |
