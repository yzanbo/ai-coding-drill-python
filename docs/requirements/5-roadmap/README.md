## このディレクトリの役割

プロジェクトの**時間軸**を扱う。

- リリース単位のロードマップ（R0〜R9）
- 優先度付きプロダクトバックログ（Now / Next / Later / Parked）
- スプリント運用ルール（DoR / DoD・セレモニー）
- 見積もり方針・リスクレジスタ

「何を作るか（What）」は他バケット（[1-vision/](../1-vision/) / [4-features/](../4-features/)）、ここは「**いつ作るか（When）**」のみ。

## 更新タイミング

- **スプリント開始時**：Now バックログから引き入れる項目を選び、DoR を確認
- **スプリント終了時**：完了項目を Now → 完了に移動、Next の見直し、ステータス更新
- **リリース完了時**：ロードマップ表の該当行を完了マークに更新
- **新規 ADR / 新規機能発生時**：バックログにリンク追加

---

## アジャイル運用との整合

- 各バックログ項目は [4-features/](../4-features/) の F-XX または独立技術タスクと紐付く
- 受け入れ条件は features/ が SSoT、ここはバックログとしての位置づけのみ
- **「いつまでに何を出すか」はコミットメントではなく Forecast**（ベロシティ × 残バックログから都度予測）
